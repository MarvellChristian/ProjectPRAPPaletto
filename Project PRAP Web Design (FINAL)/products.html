Intel 12th Gen Alder Lake Rilis dengan Performa yang Gahar, Bikin Pesaingnya Ketar-Ketir!

Halo pecinta teknologi! Di akhir tahun 2021 ini, persaingan di dunia prosesor rasanya lagi panas-panasnya, nih! Setelah bangkitnya penjualan prosesor AMD Ryzen hingga 93% pada kuartal pertama 2021, rilisnya prosesor Apple M1 Max dan M1 Pro, kini giliran Intel untuk menunjukkan jati dirinya sebagai perusahaan prosesor terbaik di dunia. Pada tanggal 27 dan 28 Oktober 2021 Intel meluncurkan prosesor terbarunya, yakni Intel Alder Lake kepada publik.


Banyak Fitur Baru yang Canggih dan Hemat Daya!

Intel baru saja meluncurkan 6 prosesor Alder Lake barunya untuk versi desktop, yakni Core i9-12900K, Core i9-12900KF, Core i7-12700K, Core i7-12700KF, Core i5-12600K, dan Core i5-12600KF. Perbedaan kode K dan KF pada produk Alder Lake adalah pada kode KF Alder Lake hadir tanpa GPU terintegrasi. Sedangkan pada kode K, terdapat GPU terintegrasi, yakni Intel UHD Graphic 770. Keenam prosesor tersebut menggunakan socket LGA 1700.

Intel Alder Lake menggunakan proses fabrikasi terbaru yang disebut ‘Intel 7’, yakni fabrikasi 10nm. Yup, berbeda dengan tradisi Intel sejak 6th gennya, intel 12th gen tidak menggunakan fabrikasi 14nm, melainkan intel 12th gen menggunakan fabrikasi 10nm sehingga efisiensi daya sangat meningkat pada Intel Alder Lake ini.

Selain sudah menggunakan fabrikasi 10nm, Intel Alder Lake juga menggunakan kombinasi core layaknya prosesor smartphone. Core pada Alder Lake terdiri dari performance cores (P-Core) dan efficient cores (E-Core).

P-Core akan aktif berjalan saat perangkat menjalankan proses yang berat dan membutuhkan daya lebih besar, misalnya bermain game AAA. Sedangkan E-Core akan aktif berjalan ketika perangkat sedang menjalankan proses atau aplikasi ringan, misalnya saat membuka dokumen, memutar lagu, dan proses-proses ringan lainnya.


Peningkatan Performa dan Mendukung RAM DDR5 serta SSD PCIe 5.0

Intel Alder Lake diklaim 19% lebih kencang dibandingkan dengan generasi pendahulunya, yaitu Intel Rocket Lake atau Intel 11th gen. Untuk mencapai performa tertinggi dari kombinasi P-Core dan E-Core dengan menggunakan Thread Director, Intel menyarankan para penggunanya untuk menggunakan sistem operasi Windows 11. Namun, Intel juga mengatakan bahwa mereka memaksimalkan performa pada Windows 10 sehingga peak performance antara Windows 11 dan Windows 10 tidak berbeda, melainkan hanya berbeda sedikit pada run-to-run variable-nya.

Performa gahar Intel Alder Lake ternyata juga mendukung penggunaan memori RAM DDR5 yang speed terendahnya berkisar pada 4800MHz. Wow! Walaupun begitu, Alder Lake juga masih mendukung RAM DDR4, namun penggunaannya sampai hari artikel ini ditulis (31 Oktober 2021) belum bisa digabungkan dengan RAM DDR5. Tidak hanya RAM DDR5, tetapi Alder Lake juga mendukung penggunaan PCIe 5.0 up to 64GB/s. Hal ini berarti di masa yang akan datang kita dapat merasakan SSD PCIe gen 5. Intel juga mengklaim bahwa Alder Lake adalah prosesor terbaik untuk gaming pada saat ini, lho!


Spesifikasi Lengkap, Pengiriman, dan Harga Alder Lake

Varian Intel Alder Lake tertinggi, yakni i9-12900K, memiliki 16 cores yang terdiri dari 8 P-Core dan 8 E-Core. P-Core pada Intel i9-12900K memiliki kecepatan base clock sebesar 3.2GHz, boost clock hingga 5.1GHz, dan maksimum hingga 5.2GHz. Sedangkan untuk E-Corenya sendiri memiliki base clock sebesar 2.4GHz dan boost clock sebesar 3.9GHz.

Varian i9, i7, dan i5 Alder Lake memiliki jumlah core, threads, cache size, clock speed, dan maksimum turbo power yang berbeda-beda. Namun, ketiga lini tersebut memiliki kesamaan pada integrated GPU-nya yakni Intel UHD Graphics 770, total CPU PCIe Lanes, dan memory channels.

Bagaimana dengan harganya? 6 prosesor baru Alder Lake dijual dengan harga:

  • Core i5-12600KF seharga Rp 3,7 Juta, Core i5-12600K seharga Rp 4,1 Juta
  • Core i7-12700KF seharga Rp 5,5 Juta, Core i7-12700K seharga Rp 5,8 Juta
  • Core i9-12900KF seharga Rp 8 Juta, dan Core i9-12900K seharga Rp 8,4 Juta
Alder Lake sedang dalam proses pre-order dan akan segera dikirim mulai tanggal 4 November 2021.

Ingin Jadi Programmer? Yuk Berkenalan dengan Git!

Halo pecinta teknologi! Berbicara tentang Git, kita perlu menyadari bahwa segala pekerjaan kita sering mengalami pembaharuan. Hal itu agar pekerjaan menghasilkan output ataupun hasil yang lebih memuaskan dan lebih efisien, bukan?

Nah, dalam karir menjadi seorang programmer, tentu kamu akan sering sekali melakukan pembaharuan, perbaikan, ataupun revisi. Seluruh perubahan dalam kode tersebut sudah seharusnya dicatat dan disimpan dengan sebaik-baiknya. Apalagi jika proyek yang kamu kerjakan adalah proyek tim, alias mengerjakan bersama-sama dengan tim. Dalam hal inilah Git berperan besar!


Version Control System

Sebelum kita masuk ke pengenalan terhadap Git, kamu harus mengetahui apa itu Version Control System (VCS) terlebih dahulu. Version Control System adalah source code management atau sistem yang mengelola perubahan dari source code.

Selain mencatat ataupun merekam perubahan yang terjadi pada source code, kita juga dapat melihat siapa yang melakukan ataupun memberikan perubahan terhadap kode dan kapan perubahan tersebut terjadi dengan menggunakan VCS. Kita juga dapat kembali ke keadaan sebelum terjadinya perubahan (checkout), lho!

Kamu pernah melakukan revisi pada tugas seperti gambar di atas? Kurang lebih begitulah yang dikerjakan oleh VCS, tetapi dengan lebih lengkap dan efisien. Perubahan dicatat dalam 1 file yang mengalami perubahan-perubahan itu sendiri dalam foldernya (repository/repo). Sehingga, untuk mencatat perubahan pada file, kita tidak perlu membuat file baru setiap kita melakukan perubahan.


Git

Git adalah software berbasis Version Control System yang digunakan untuk mengelola perubahan file di dalam folder (repository). Riwayat perubahan yang terjadi pada repo disimpan menggunakan rangkaian commit. Di dalam commit kita juga dapat melihat hash atau penanda commit, editor, kapan perubahan terjadi, dan commit message.

Kita juga bisa membuat cabang dari repository untuk mengerjakan dan mengembangkan proyek lain tanpa mempengaruhi repository pusat. Cabang ini disebut juga dengan istilah branch. Dengan adanya branch, pekerjaan tim akan sangat terbantu karena setiap anggota tim dapat mengerjakan pekerjaannya masing-masing tanpa merusak ataupun mengganggu pekerjaan sesama rekan tim, kemudian hasil pekerjaan setiap anggota tim dapat digabung lagi ke file utama setelah pekerjaan masing-masing anggota selesai (merge).

GitHub

Ada Git, ada GitHub? Apa itu GitHub? Apakah sama dengan Git? Mungkin pertanyaan itu ada di benak kamu sekarang. Jawabannya, GitHub adalah sebuah website yang menjadi layanan cloud untuk menyimpan dan mengelola repository git. GitHub dan Git tidaklah sama. Kamu bisa saja menginstall dan menggunakan Git tanpa menggunakan GitHub. Sebaliknya, kamu juga bisa menggunakan GitHub tanpa meng-install Git di komputer kamu. Akan tetapi, mengombinasikan pemakaian Git dan GitHub jauh lebih baik dan kebanyakan programmer pasti melakukannya.

Kita bisa melakukan hal yang sama seperti yang kita lakukan di local machine atau di software Git kita dengan menggunakan GitHub, seperti misalnya membuat repo, melakukan commit, branch, dan merge. Namun, bedanya kita melakukannya secara online dengan sistem cloud. Jadi, bisa dikatakan bahwa GitHub adalah website yang mengandung git.

Dengan menggunakan keduanya baik Git maupun GitHub, kita bisa mengirim source code atau proyek kita dari komputer kita ke GitHub (push), dan sebaliknya kita juga bisa mengambil source code, proyek, ataupun repo dari GitHub ke komputer kita (pull). Sesuatu yang kita pull dan/atau push itu berupa commit. Bagaimana agar kita bisa melakukannya? Kita harus menjadikan GitHub sebagai remote yaitu dengan cara membuat repo di GitHub dan meng-clone repo tersebut ke komputer kita.

Dengan sistem seperti ini, kita akan dipermudah saat mengerjakan suatu proyek bersama tim. Karena kita bisa mengerjakan suatu repo yang satu pada GitHub, dan kita membagikan tugas masing-masing dan mengerjakannya di komputer masing-masing. Layanan seperti ini tidak hanya GitHub, melainkan ada juga Bitbucket dan GitLab.


Kesimpulan

  • Version Control System adalah sistem yang mengelola perubahan dari source code.
  • Git adalah salah satu software VCS.
  • GitHub adalah salah satu website atau layanan untuk mengelola proyek Git.
